# 免费部署方案

针对您的量化策略回测平台（Flask后端 + Vue前端），以下是推荐的免费部署方案：

## 方案一：Render（推荐）⭐⭐⭐⭐⭐

**适合场景**：全栈应用（Python后端 + Vue前端）

### 优势
- ✅ **完全免费**：提供永久免费实例
- ✅ 支持Python（Flask）和前端
- ✅ 自动部署：连接GitHub仓库后自动构建部署
- ✅ SSL证书：自动配置HTTPS
- ✅ 无需配置域名：提供免费子域名

### 部署步骤

#### 1. 准备项目文件
在项目根目录创建以下文件：

**`requirements.txt`**（已有）
```
Flask==3.0.0
Flask-CORS==4.0.0
pandas==2.1.0
numpy==1.24.3
akshare>=1.16.72
scipy==1.11.4
python-dateutil==2.8.2
pytz==2023.3
gunicorn==21.2.0
```

**`runtime.txt`**（新增）
```
python-3.9.16
```

**`Procfile`**（新增）
```
web: gunicorn app:app --host 0.0.0.0 --port $PORT
```

#### 2. 部署后端
1. 访问 https://render.com 注册账号
2. 点击 "New +" → "Web Service"
3. 连接GitHub账号并选择您的项目仓库
4. 配置构建和启动：
   - **Root Directory**: `backend`
   - **Build Command**: `pip install -r ../requirements.txt`
   - **Start Command**: `gunicorn app:app --host 0.0.0.0 --port $PORT`
   - **Python Version**: `3.9.16`
5. 点击 "Create Web Service"

#### 3. 部署前端
1. 点击 "New +" → "Static Site"
2. 选择同一个GitHub仓库
3. 配置：
   - **Root Directory**: `frontend`
   - **Build Command**: `npm run build`
   - **Publish Directory**: `dist`
   - **API代理配置**：在 `vite.config.js` 中修改API地址为Render后端URL

### 免费额度
- Web服务：512MB RAM，0.1 CPU
- 每月750小时
- 数据库：90天免费PostgreSQL（可选）

### 注意事项
- 免费实例15分钟无请求会休眠，首次请求需要30-60秒唤醒
- 不建议用于高频交易，适合个人研究和回测
- AkShare数据请求可能较慢，建议缓存数据

## 方案四：混合部署方案（最经济）⭐⭐⭐⭐⭐

**前端**：Vercel（免费）
**后端**：Render（免费）

### 优势
- ✅ 前端使用Vercel，全球CDN加速
- ✅ 后端使用Render，支持Python
- ✅ 两者都完全免费
- ✅ 部署速度快，体验好

### 部署步骤

#### 1. 前端部署到Vercel
```bash
# 1. 前端配置修改
cd frontend
# 修改 vite.config.js 的 API 代理为实际后端地址
```

访问 https://vercel.com 注册并导入GitHub项目即可自动部署。

#### 2. 后端部署到Render
按照方案一的步骤部署后端。

#### 3. 配置CORS
后端需要配置允许前端域名访问：
```python
# backend/app.py
from flask_cors import CORS
CORS(app, origins=["https://your-vercel-domain.vercel.app"])
```

## 针对您的项目的特别建议

### 方案一：Vercel + Render（强烈推荐）

**原因**：
1. 前端部署在Vercel，访问速度极快
2. 后端部署在Render，支持Python和Flask
3. 两者都永久免费，且自动部署
4. 量化策略回测不需要实时高频访问，休眠机制影响不大

### 部署清单

#### 前端准备
1. 修改 `frontend/vite.config.js`，移除dev代理配置
2. 在 `frontend/.env.production` 添加后端API地址
3. 推送代码到GitHub

#### 后端准备
1. 创建 `Procfile`
2. 添加 `runtime.txt`
3. 配置CORS允许前端域名
4. 推送代码到GitHub

#### 部署
1. 前端：导入Vercel，选择 `frontend` 目录
2. 后端：导入Render，选择 `backend` 目录
3. 获取后端URL，配置到前端环境变量
4. 重新部署前端

## 部署后的优化建议

### 1. 数据缓存
由于免费实例有限制，建议添加数据缓存：
```python
# backend/app.py
from flask_caching import Cache
cache = Cache(app, config={'CACHE_TYPE': 'simple'})

@app.route('/api/etf_pool')
@cache.cached(timeout=3600)  # 缓存1小时
def get_etf_pool():
    # ...
```

### 2. 异步任务
对于耗时的回测操作，使用后台任务：
```python
from flask import jsonify
from threading import Thread

def run_backtest_task(params, callback):
    result = run_backtest_logic(params)
    callback(result)

@app.route('/api/backtest', methods=['POST'])
def run_backtest():
    # 启动后台任务
    thread = Thread(target=run_backtest_task, args=(params, save_result))
    thread.start()
    return jsonify({'status': 'processing', 'task_id': task_id})
```

### 3. 错误处理
添加友好的错误提示：
```python
@app.errorhandler(500)
def handle_error(e):
    return jsonify({
        'status': 'error',
        'message': '服务暂时不可用，请稍后重试'
    }), 500
```

### 4. 日志记录
```python
import logging
logging.basicConfig(filename='app.log', level=logging.INFO)
```
